Steam Social Network Analysis

Friend Prediction via Shared Game Libraries · Comprehensive Analysis Report

Author: Su Xinyi

1. Project Overview

This project analyzes the Steam gaming social network by constructing a bipartite graph of 500 players and 17,182 games from five gaming communities. Using structural, graph-based, and behavioral features, we predict friendships from shared game libraries and identify cross-genre bridge players who connect otherwise disparate gaming communities. The pipeline spans data collection via the Steam Web API, feature engineering, homophily analysis, link prediction across six heuristic and three machine learning methods, and a Graph Autoencoder (GAE) achieving 0.9664 AUC-ROC.

3. Feature Engineering

Eight features were computed for every player pair (124,750 rows) from the bipartite graph and genre mapping:

FeatureCategoryDescription
Jaccard SimilarityStructuralShared games / total unique games
Weighted JaccardStructuralMin/Max playtime ratio on shared games
Common NeighborsStructuralNumber of shared games (bipartite degree)
Cosine SimilarityStructuralBinary vector cosine on game sets
Adamic-AdarGraph-basedSum of 1/log(degree) over shared games
Preferential AttachmentGraph-basedProduct of player bipartite degrees
Playtime CorrelationBehavioralPearson r on shared game hours (≥3 games)
Genre OverlapBehavioralGenre-set Jaccard via keyword heuristic

Genre Mapping

A keyword heuristic was applied to game names to assign primary genres, covering ~43% of edges and ~53% per-player. The genre overlap feature was computed as Jaccard similarity on genre sets for each player pair, replacing the initial stub value of -1.0.

Key Insight: Preferential Attachment produces raw values reaching ~22M; log-transforming (np.log1p) before feeding to classifiers avoids sklearn overflow warnings. Playtime Correlation is near-zero for most pairs since few share ≥3 games with non-zero variance.

7. Practical Deployment

Friend Recommendation System

The projected player-player graph with Jaccard, Common Neighbors, and Adamic-Adar scores serves as the foundation for a friend recommendation engine. The filter_projected_graph.py script applies configurable thresholds (e.g., Jaccard ≥ 0.05, common neighbors ≥ 5) to prune low-similarity edges and exports a GEXF graph suitable for Gephi visualization or downstream recommender logic. In a real Steam community, this could power a "Players Like You" feature — recommending friends based on shared game libraries and playtime patterns.

Cross-Genre Influencer Detection

The bridge player pipeline (bridge_players.py) identifies users with high genre entropy who naturally connect different gaming communities. These players are valuable targets for community management, cross-promotion, or influencer outreach. The ego-network clustering further validates which bridge players truly span distinct genre clusters versus simply owning many games in the same genres. A real-world deployment could automate this detection and feed results into a CRM or community analytics dashboard.

GAE-Based Prediction Service

The best-performing model (GAE, 0.9664 AUC-ROC) could be operationalized as a prediction API. New players arriving on the platform would have their game libraries encoded, fed through the trained GAE (using pre-computed Node2Vec embeddings as node features), and receive ranked friend suggestions ranked by the reconstructed adjacency scores.

8. Conclusions

Key Takeaways

  • GAE dominates: Graph Autoencoders with Node2Vec features achieve 0.9664 AUC-ROC, far exceeding heuristic and ML baselines. The GAE is the only model that produces meaningful Precision@k results (0.90 P@10).
  • Homophily exists: Friend pairs have ~54% higher mean Jaccard game similarity (0.122 vs 0.079), confirming that shared game libraries are a meaningful signal for friendship.
  • Bridge players connect communities: 50 high-entropy bridge players were identified, with 2 confirmed as cross-genre connectors via ego-network clustering.
  • Heuristic methods cap at ~0.83 AUC-ROC: Adamic-Adar and Common Neighbors perform best among unsupervised methods but plateau well below learned approaches.

Limitations

  • Extreme class imbalance (1:997) — Precision@k is near-zero for non-GAE methods; even Logistic Regression finds 0 friends in the top 50 predictions.
  • Genre coverage ~43% — The keyword heuristic leaves over half of game edges unlabeled, limiting the genre overlap feature's discriminative power.
  • Dataset scope: 500 players from 5 gaming groups may not generalize to the broader Steam population (~120M monthly active users).
  • Friendship sparsity: Only 125 friend edges exist, making cross-validation splits noisy and limiting statistical power for subgroup analysis.

Future Work

  • Enrich genre coverage using the Steam API appdetails endpoint (with rate-limit mitigation) to label all 17,182 games.
  • Explore temporal features (when games were played) and social graph features (triadic closure) beyond shared game libraries.
  • Deploy the GAE as a real-time recommendation service with incremental training for new players.
  • Extend bridge player analysis with network centrality measures (betweenness, closeness) on the projected graph.
  • Collect a larger, more diverse sample across gaming communities to validate generalizability.
